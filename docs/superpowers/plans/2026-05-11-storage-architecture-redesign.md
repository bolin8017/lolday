# Storage Architecture Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate MLflow artifacts, Harbor container blobs, and Loki log chunks from K3s local-path-provisioner PVCs to a self-hosted MinIO S3-compatible object store, with operator-friendly SSD expansion runbook and per-bucket lifecycle policies.

**Architecture:** Add MinIO as a Helm sub-chart under `charts/lolday/`, deployed as a single-node-single-drive StatefulSet bound to a 100 Gi local-path PVC. An init Job pre-creates buckets, IAM service accounts (one per consuming application), and lifecycle policies. Each application's Helm template/values are flipped from filesystem driver to S3 driver pointing at `minio.lolday.svc:9000`. SSD expansion uses MinIO's "server pool" pattern — each new disk becomes a new pool added to the StatefulSet, zero rebalance, zero downtime.

**Tech Stack:** Helm 3 (umbrella chart with sub-charts), MinIO server (Single-Node Single-Drive mode), `mc` (MinIO admin client) for init job, K8s Secret + StatefulSet + Job, existing `bash scripts/deploy.sh` deploy entrypoint. Tests: `helm lint`, `helm template` diff, `kubectl wait`, MinIO Prometheus metrics scrape, end-to-end smoke against MLflow / Harbor / Loki API.

Reference spec: `docs/superpowers/specs/2026-05-11-storage-architecture-redesign-design.md`

### Task 3: MinIO bucket + service-account init Job

**Files:**

- Create: `charts/lolday/templates/minio-init-buckets-job.yaml`

This Job runs once per Helm install/upgrade and performs all storage-layer initialization:

1. Waits for MinIO service ready
2. Creates 4 buckets: `mlflow-artifacts` (versioned), `harbor-blobs`, `loki-chunks`, `loki-ruler`
3. Creates 3 IAM policies (one per app, scoped to its bucket(s))
4. Creates 3 service accounts: `mlflow-app`, `harbor-app`, `loki-app` (only if not already present)
5. Writes the service-account credentials to 3 K8s secrets: `mlflow-s3-cred`, `harbor-s3-cred`, `loki-s3-cred` (only if not already present)
6. Applies lifecycle rule: 7-day object expiration on `loki-chunks`

Design: the Job uses an **initContainer + main container** pattern because the `quay.io/minio/mc` image has no `kubectl`, and the `bitnami/kubectl` image has no `mc`. The two containers share an `emptyDir` volume `/creds`:

- initContainer (`mc-setup`) — provisions buckets / policies / service accounts in MinIO; writes generated credentials to `/creds/<app>-access-key` and `/creds/<app>-secret-key`
- main container (`kubectl-secret-writer`) — reads `/creds/*`, creates K8s secrets only if they don't already exist (idempotent)

- [ ] **Step 1: Write the full Job template**

Create `charts/lolday/templates/minio-init-buckets-job.yaml`:

```yaml
{{- if .Values.minio.enabled }}
apiVersion: v1
kind: ServiceAccount
metadata:
  name: minio-init-buckets
  namespace: {{ .Release.Namespace }}
  annotations:
    "helm.sh/hook": post-install,post-upgrade
    "helm.sh/hook-weight": "-1"
---
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: minio-init-buckets
  namespace: {{ .Release.Namespace }}
  annotations:
    "helm.sh/hook": post-install,post-upgrade
    "helm.sh/hook-weight": "-1"
rules:
  - apiGroups: [""]
    resources: ["secrets"]
    verbs: ["get", "create"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: minio-init-buckets
  namespace: {{ .Release.Namespace }}
  annotations:
    "helm.sh/hook": post-install,post-upgrade
    "helm.sh/hook-weight": "-1"
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: Role
  name: minio-init-buckets
subjects:
  - kind: ServiceAccount
    name: minio-init-buckets
    namespace: {{ .Release.Namespace }}
---
apiVersion: batch/v1
kind: Job
metadata:
  name: minio-init-buckets
  namespace: {{ .Release.Namespace }}
  labels:
    app.kubernetes.io/managed-by: helm
    app.kubernetes.io/part-of: lolday-storage
  annotations:
    "helm.sh/hook": post-install,post-upgrade
    "helm.sh/hook-weight": "0"
    "helm.sh/hook-delete-policy": before-hook-creation,hook-succeeded
spec:
  backoffLimit: 3
  ttlSecondsAfterFinished: 3600
  template:
    spec:
      restartPolicy: OnFailure
      serviceAccountName: minio-init-buckets
      volumes:
        - name: creds
          emptyDir: {}
      initContainers:
        - name: mc-setup
          image: quay.io/minio/mc:RELEASE.2025-09-07T16-13-09Z
          env:
            - name: MINIO_ROOT_USER
              valueFrom: { secretKeyRef: { name: minio-root-cred, key: rootUser } }
            - name: MINIO_ROOT_PASSWORD
              valueFrom: { secretKeyRef: { name: minio-root-cred, key: rootPassword } }
          volumeMounts:
            - { name: creds, mountPath: /creds }
          command:
            - /bin/sh
            - -c
            - |
              set -eux
              # Wait for MinIO API ready
              until mc alias set local http://minio:9000 "$MINIO_ROOT_USER" "$MINIO_ROOT_PASSWORD" 2>/dev/null; do
                echo "waiting for minio..."
                sleep 2
              done

              # Idempotent bucket creation
              mc mb --ignore-existing local/mlflow-artifacts
              mc mb --ignore-existing local/harbor-blobs
              mc mb --ignore-existing local/loki-chunks
              mc mb --ignore-existing local/loki-ruler

              # MLflow artifact versioning (best-effort — single-drive mode doesn't support it but the spec calls for it once multi-pool is online)
              mc version enable local/mlflow-artifacts || true

              # Lifecycle: 7-day expiration on Loki chunks
              cat > /tmp/loki-lifecycle.json <<JSON
              {"Rules":[{"ID":"loki-7d-expire","Status":"Enabled","Expiration":{"Days":7},"Filter":{"Prefix":""}}]}
              JSON
              mc ilm import local/loki-chunks < /tmp/loki-lifecycle.json

              # Per-app IAM policies — separate JSON per app to keep them readable
              cat > /tmp/mlflow-rw.json <<'JSON'
              {"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":["s3:*"],
                "Resource":["arn:aws:s3:::mlflow-artifacts","arn:aws:s3:::mlflow-artifacts/*"]}]}
              JSON
              cat > /tmp/harbor-rw.json <<'JSON'
              {"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":["s3:*"],
                "Resource":["arn:aws:s3:::harbor-blobs","arn:aws:s3:::harbor-blobs/*"]}]}
              JSON
              cat > /tmp/loki-rw.json <<'JSON'
              {"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":["s3:*"],
                "Resource":["arn:aws:s3:::loki-chunks","arn:aws:s3:::loki-chunks/*",
                            "arn:aws:s3:::loki-ruler","arn:aws:s3:::loki-ruler/*"]}]}
              JSON
              for app in mlflow harbor loki; do
                mc admin policy create local ${app}-rw /tmp/${app}-rw.json || \
                  mc admin policy update local ${app}-rw /tmp/${app}-rw.json
              done

              # Per-app service accounts — generate access/secret key and persist to shared volume.
              # `mc admin user svcacct add` errors if the access key already exists, so we
              # generate fresh keys only on first install (no /creds/done sentinel exists).
              if [ ! -f /creds/done ]; then
                for app in mlflow harbor loki; do
                  AK=$(tr -dc 'a-zA-Z0-9' </dev/urandom | head -c 20)
                  SK=$(tr -dc 'a-zA-Z0-9' </dev/urandom | head -c 40)
                  mc admin user svcacct add local "$MINIO_ROOT_USER" \
                    --access-key "$AK" --secret-key "$SK" --policy ${app}-rw
                  printf '%s' "$AK" > /creds/${app}-access-key
                  printf '%s' "$SK" > /creds/${app}-secret-key
                done
                touch /creds/done
              fi
              echo "mc-setup: done"
      containers:
        - name: kubectl-secret-writer
          image: bitnami/kubectl:1.30.6
          volumeMounts:
            - { name: creds, mountPath: /creds, readOnly: true }
          command:
            - /bin/sh
            - -c
            - |
              set -eux
              # Skip creating K8s secrets that already exist (preserves credentials
              # if init-job re-runs from a Helm upgrade; the apps mid-flight keep
              # using the unchanged secret).
              if [ ! -f /creds/done ]; then
                echo "no creds to write (already present from prior install)"
                exit 0
              fi
              for app in mlflow harbor loki; do
                if kubectl -n {{ .Release.Namespace }} get secret ${app}-s3-cred >/dev/null 2>&1; then
                  echo "${app}-s3-cred already exists — keep"
                  continue
                fi
                kubectl -n {{ .Release.Namespace }} create secret generic ${app}-s3-cred \
                  --from-file=access-key=/creds/${app}-access-key \
                  --from-file=secret-key=/creds/${app}-secret-key
              done
              echo "kubectl-secret-writer: done"
{{- end }}
```

> **Idempotence model**: on re-runs (Helm upgrade post-hook), `mc-setup` re-asserts buckets, policies, and lifecycle (all `--ignore-existing` or upsert-style); `mc admin user svcacct add` is skipped via the `/creds/done` sentinel; and `kubectl-secret-writer` skips writes when the K8s secret already exists. Net effect: zero-op after first successful install.

### Task 6: Migrate existing MLflow artifact data to S3

**Files:**

- Create: `scripts/migrate-mlflow-to-s3.sh`

- [ ] **Step 1: Write the migration script**

Create `scripts/migrate-mlflow-to-s3.sh`:

```bash
#!/usr/bin/env bash
# One-shot MLflow artifact migration: PVC filesystem → MinIO S3.
# Idempotent (uses `mc mirror`, skips equal files by ETag).
#
# Spec: docs/superpowers/specs/2026-05-11-storage-architecture-redesign-design.md §6.5
set -euo pipefail

NS=lolday
MINIO_POD="$(kubectl get pod -n $NS -l app.kubernetes.io/name=minio -o name | head -1)"

echo "==> Wait for MinIO ready"
kubectl rollout status -n $NS statefulset/minio --timeout=2m

echo "==> Configure mc alias in MinIO pod"
ROOT_USER=$(kubectl get secret -n $NS minio-root-cred -o jsonpath='{.data.rootUser}' | base64 -d)
ROOT_PASS=$(kubectl get secret -n $NS minio-root-cred -o jsonpath='{.data.rootPassword}' | base64 -d)
kubectl exec -n $NS $MINIO_POD -- mc alias set local http://minio:9000 "$ROOT_USER" "$ROOT_PASS"

echo "==> Find existing MLflow pod (it still has /mlflow-artifacts PVC mounted pre-rollout)"
MLFLOW_POD="$(kubectl get pod -n $NS -l app.kubernetes.io/component=mlflow -o name | head -1)"

echo "==> Inventory: how many files in /mlflow-artifacts?"
kubectl exec -n $NS $MLFLOW_POD -- find /mlflow-artifacts -type f | wc -l

echo "==> Copy via temporary tar-stream + mc cp (avoids in-pod mc install)"
# Pipe a tar of /mlflow-artifacts through stdout, then write into MinIO via mc pipe.
kubectl exec -n $NS $MLFLOW_POD -- tar cf - -C /mlflow-artifacts . | \
  kubectl exec -i -n $NS $MINIO_POD -- /bin/sh -c '
    set -eux
    mkdir -p /tmp/mlflow-import
    tar xf - -C /tmp/mlflow-import
    mc mirror /tmp/mlflow-import/ local/mlflow-artifacts/
    rm -rf /tmp/mlflow-import
  '

echo "==> Verify object count in S3 matches FS"
FS_COUNT=$(kubectl exec -n $NS $MLFLOW_POD -- find /mlflow-artifacts -type f | wc -l)
S3_COUNT=$(kubectl exec -n $NS $MINIO_POD -- mc ls --recursive local/mlflow-artifacts/ | wc -l)
```

```markdown
## 6. Storage architecture

lolday runs a layered storage strategy:

| Layer                            | Backend                                                | Used by                                                                                                 |
| -------------------------------- | ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------- |
| **Object store** (S3-compatible) | MinIO single-node, scale via multi-pool SSDs           | MLflow artifacts, Harbor container blobs, Loki log chunks                                               |
| **OLTP / TSDB (block)**          | K3s local-path-provisioner on NVMe                     | PostgreSQL (lolday + mlflow + harbor metadata), Prometheus TSDB, Grafana, Alertmanager, Trivy DB, Redis |
| **Read-only big static**         | hostPath at `/mnt/ssd500g/data/samples` (separate SSD) | Malware samples (600 GB ROX)                                                                            |

### 6.1 Why this split

- **Object** for write-once-read-many, append-mostly, large file workloads — fits MLflow models, container layers, log chunks
- **Block** for transactional small writes, mmap, page-cache-friendly workloads — fits PostgreSQL and Prometheus
- **Read-only hostPath** for big static input data — avoids copying it through PVs

### 6.2 SSD expansion

Adding capacity = mount new SSD + add as new MinIO server pool. See `docs/runbooks/add-ssd.md`. The MinIO multi-pool approach gives **zero-downtime, zero-rebalance** SSD addition. Application config (MLflow / Harbor / Loki) is not touched.

### 6.3 Multi-node future

MinIO supports distributed mode (multi-node-multi-drive). When a second server joins, MinIO transitions to distributed deployment using the same S3 endpoint. **No application changes needed**. See spec §5.6.

### 6.4 Retention / lifecycle

- MLflow artifacts: **versioned, no auto-expiration** (model is first-class asset)
- Harbor blobs: cleaned by **Harbor's own GC** (do not set MinIO lifecycle — see Harbor docs)
- Loki chunks: **7-day MinIO lifecycle** (object expiration)

### 6.5 Backup

**Currently**: not implemented. Server30 single point of failure. Mainstream pattern is MinIO site-replication to a backup site; future scope when a backup node exists.

### 6.6 Capacity thresholds & alerts

Discord alert (#lolday-service-alerts) triggers when:

- MinIO cluster free < 10 GB
- Any individual bucket → projected growth crosses spec §5.4 thresholds

Alert source: `minio_cluster_capacity_usable_free_bytes` Prometheus metric.

### Tech debt — vcjob TTL

The `lolday-controllers` Deployment doesn't honor vcjob `ttlSecondsAfterFinished` (no Volcano controller-manager running). Stale vcjobs accumulate, currently cleared manually. **Follow-up spec planned.**
```
